# 每日打卡Pro - UI设计方案

> 设计理念：**「极简·温暖·激励」**
> - 极简：1步打卡，零学习成本
> - 温暖：社交陪伴，不孤单的坚持
> - 激励：即时反馈，看得见的成长

---

## 一、设计语言与色彩系统

### 1.1 核心色彩

| 色彩名称 | 色值 | 应用场景 | 情感传达 |
|---------|------|---------|---------|
| **主色** | `#6C5CE7` (活力紫) | 主按钮、进度条、品牌标识 | 成长、蜕变、坚持 |
| **成功色** | `#00B894` (清新绿) | 已打卡状态、完成目标 | 成就、满足 |
| **警告色** | `#FDCB6E` (温暖黄) | 补签、提醒、连续中断 | 温和提醒、补救机会 |
| **未完成色** | `#DFE6E9` (中性灰) | 未打卡状态 | 待完成、中性 |
| **强调色** | `#FF7675` (活力红) | 限时挑战、紧急提醒 | 紧迫感、行动力 |
| **背景色** | `#F8F9FA` (浅灰白) | 页面背景 | 干净、舒适 |
| **卡片背景** | `#FFFFFF` (纯白) | 卡片、内容区 | 清晰、聚焦 |

### 1.2 渐变系统

```
品牌渐变: linear-gradient(135deg, #6C5CE7 0%, #a29bfe 100%)
成功渐变: linear-gradient(135deg, #00B894 0%, #55EFC4 100%)
警告渐变: linear-gradient(135deg, #FDCB6E 0%, #FFEAA7 100%)
挑战渐变: linear-gradient(135deg, #FF7675 0%, #FD79A8 100%)
```

### 1.3 字体规范

| 场景 | 字号 | 字重 | 行高 | 颜色 |
|-----|------|------|------|------|
| 页面标题 | 36rpx | Bold | 48rpx | #2D3436 |
| 卡片标题 | 32rpx | SemiBold | 44rpx | #2D3436 |
| 正文内容 | 28rpx | Regular | 40rpx | #636E72 |
| 辅助文字 | 24rpx | Regular | 36rpx | #B2BEC3 |
| 按钮文字 | 30rpx | SemiBold | - | #FFFFFF |
| 数字强调 | 40rpx | Bold | - | #6C5CE7 |

## 三、组件库设计

### 3.1 按钮组件

| 组件名称 | 尺寸 | 颜色 | 用途 |
|---------|------|------|------|
| **主按钮** | 高度88rpx | 紫色渐变 | 主要操作（确认、打卡） |

## 六、动效设计

### 6.1 首次打开动画

```
1. Logo淡入（0-300ms）
2. 标题"每日打卡Pro"从下方滑入（300-600ms）
3. 底部Tab栏从下方滑入（600-900ms）
4. 首页内容依次淡入（900-1200ms）
```

### 6.2 打卡成功动画

```
1. 打卡按钮变为绿色勾选图标（0-200ms）
2. 勾选图标放大1.2倍（200-400ms）
3. 恢复原大小+震动反馈（400-500ms）
4. Toast从下方滑入"打卡成功+5积分"（500-800ms）
```

### 6.3 获得勋章动画

```
1. 勋章从中心放大出现（0-300ms）
2. 粒子效果从勋章向外扩散（300-600ms）
3. "恭喜获得XX勋章"文字从下方滑入（600-900ms）
4. "分享"按钮淡入（900-1000ms）
```

### 6.4 挑战完成动画

```
1. 彩带从顶部飘落（0-1000ms）
2. "恭喜完成挑战！"文字放大出现（500-800ms）
3. 积分数字滚动增长（800-1500ms）
4. "获得双倍积分"文字淡入（1500-1800ms）
```

## 十、设计原则总结

1. **极简至上**：1步打卡，零学习成本
2. **即时反馈**：每次操作都有明确反馈
3. **激励驱动**：积分、勋章、挑战多重激励
4. **社交陪伴**：打卡圈、组队、排行榜
5. **温暖亲切**：鼓励文案、柔和动画、友好提示
6. **性能优先**：轻量化设计，快速加载
7. **无障碍**：适配不同用户需求
